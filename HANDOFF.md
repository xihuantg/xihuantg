# 豫志通项目交接文档

更新时间：2026-06-04

## 1. 项目概况

项目名：豫志通

定位：面向河南新高考 3+1+2 考生和家长的志愿填报辅助工具。当前是 GitHub Pages 静态前端展示版，不接真实后端、不接真实支付、不接真实 AI API。

目标用户：河南优先，后续可扩展山东、河北、山西等山河四省家庭。

核心卖点：

- 根据河南 3+1+2 选科、分数、位次、批次、城市和专业偏好生成冲稳保推荐。
- 不只给学校名单，还解释选科、位次、专业组风险、就业、考研、学习难度、转专业路径。
- 低价线上报告型产品，不做线下一对一专家服务。
- 重点解决普通家庭怕浪费分、怕滑档、怕专业坑、家里讨论不清楚的问题。

## 2. 当前功能

### 桌面端

- 首页诊断表单：河南、首选科目、再选科目、分数、位次、批次、专业偏好、城市偏好。
- 智能推荐：按冲、稳、保、不可报展示院校专业组；桌面端保留统计条和推荐卡片。
- 志愿表：可加入、排序、删除，免费版限制数量。
- 风险诊断：检查选科不匹配、保底不足、冲刺过多、热门专业集中等问题。
- 金榜专业组榜单：院校优先 / 专业优先切换外观，展示考生档案、风险分桶、筛选条、院校专业组卡片、近年录取数据、专业方向和操作按钮。
- 专业目录：93 个专业类，支持搜索和门类筛选。
- 学校详情页：官网、校园照片占位、录取趋势、选科要求、就业升学、适合画像、学习难度、转专业与校内路径。
- 会员套餐：基础版 29 元 / 本届高考季，进阶版 59 元 / 本届高考季。
- AI 性格与职业倾向测评：10 道选择题，生成画像、适合方向、专业和院校专业组推荐。

### 手机端

- App 风格首页。
- 固定底部导航：首页、推荐、院校、诊断、我的。
- 首页工具入口：智能推荐、性格测评、志愿诊断、院校库、专业目录、会员权益。
- 推荐页顶部已压缩成可交互工具条：筛选按钮直接显示“全部/冲/稳/保/不可报”的数量，不再用大面积说明和统计卡片占首屏。
- 院校页使用高密度金榜专业组卡片，接近快志愿截图的扫读体验。
- 性格测评归入诊断页。
- 推荐学校和院校榜单都可进入详情页。

### 桌面/手机识别

最新主干已加入设备识别逻辑：

- 桌面浏览器保持桌面布局，即使窗口较窄也不强行切成手机端。
- 手机端根据 UA、触控、屏幕尺寸、hover/pointer 能力综合判断。
- 相关提交：`eb5b65b Keep desktop layout on desktop devices`。

## 3. 视觉方向

当前新版走“金榜国风”路线：

- 关键词：金榜题名、朱红、墨黑、鎏金、暖纸底、榜单边框、强冲刺感。
- 目的：报志愿是一件激动的事，视觉上要比早期清新配色更有仪式感和决策氛围。
- 保持可读性：正文仍用易读中文黑体字体栈，标题更偏仪式感中文衬线字体栈，不引入外部字体依赖。
- 手机端尽量高密度、强操作，不做大面积无交互说明。

注意：不要再回到淡蓝、清新、轻工具感的一套配色。

## 8. 重要注意事项

### 编码问题

之前出现过中文变成 `????` 或乱码的问题。

原因：在 Windows PowerShell 里通过管道或脚本直接写中文，编码可能被转换，导致写入文件前中文已经损坏。

以后改中文时：

- 优先用 `apply_patch`。
- 如果必须脚本写文件，脚本文件本身要用 UTF-8 保存，再用 Python 读取/写入 UTF-8。
- 不要用 PowerShell 管道直接写大段中文。
- 修改后必须扫描冲突标记和乱码。

建议扫描：

```powershell
node --check app.js
node --check data.js
git diff --check
git grep -n -E '(<{7}|>{7}|\\xEF\\xBF\\xBD)' -- index.html data.js app.js styles.css HANDOFF.md
Select-String -Path index.html,data.js,app.js,styles.css,HANDOFF.md -Pattern '\?\?+'
```

### 数据真实性

当前所有院校、分数、位次、就业、环境评价、转专业说明都是展示版样板数据。

正式收费前必须替换为真实、可核验、可合法使用的数据，并补充来源：

- 河南省教育考试院。
- 高校招生章程。
- 高校本科生手册 / 教务处转专业政策。
- 高校官网。
- 合法可引用的公开评价摘要。

### 不做的内容

当前阶段不做：

- 真实支付。
- 真实登录。
- 后端数据库。
- 真实 AI API。
- 线下一对一专家服务。
- 保录取、内部数据、百分百准确等承诺。
